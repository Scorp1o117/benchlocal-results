# BenchLocal-Results V1.0.0

BenchLocal-Results V1.0 是项目第一次完整版本更新。本次发布不只是重新设计页面，而是重构了从 BenchLocal 测试结果到可发布网站的整条链路：原始运行匹配、规范化快照、双评分、双模式合并、静态页面生成和发布前校验现在由同一套流程管理。

## 本次更新的核心目标

旧版站点能够展示成绩，但数据、模式和页面之间仍需要较多人工核对。V1.0 将规范化模型快照设为网站的唯一数据源，让每个页面上的分数都能回溯到对应的 ToolCall、BugFind 和 HermesAgent 运行结果，同时减少重复卡片和重复详情页。

## 数据与评分

- 收录 **15 个模型、23 个测试配置**，其中 **8 个模型**拥有可直接比较的思考与无思考结果。
- 从 BenchLocal `runs/` 或 `by-model/` 匹配三套测试结果，并记录 run ID、原始 summary 路径和 SHA-256。
- 能力上限按 `ToolCall × 0.3 + BugFind × 0.3 + HermesAgent × 0.4` 计算。
- 实用得分为能力上限减去成功题的额外重试次数；首次通过不扣分，失败题不会重复扣除重试成本。
- 详情页保留三套题集共 50 个场景的最终状态、分数、首次结果、尝试次数和重试次数。
- 分数统一格式化，修复浮点运算产生的长尾小数显示。

## 思考与无思考模式

- 同一模型的 thinking 与 no-thinking 结果合并为一张首页卡片。
- 排行榜支持全部模型、思考、无思考筛选，并根据当前模式重新计算排序指标。
- 详情页在同一入口展示两种模式，可切换查看专项成绩、实用得分、重试和完整题目明细。
- 页面自动生成模式差异，包括能力上限、实用得分、额外重试以及 TC / BF / HA 专项差值。

## 自动化发布流程

测试完成后的标准流程为：

```powershell
npm run status
npm run extract -- --latest --variant <thinking|no-thinking>
npm run build
npm run check
```

- `status` 检查最新三套测试是否属于同一模型，并确认 TC → BF → HA 的完成顺序。
- `extract` 验证运行、提取最终分数和逐题结果、统计重试、生成快照并校验归档哈希。
- `build` 生成目录数据、模式对比、合并首页、统一详情页和旧链接跳转页。
- `check` 重新计算评分并检查快照、页面、链接、本地资源、脚本语法和生成结果一致性。
- GitHub Actions 在 pull request 和 `main` 分支更新时自动运行验证门禁。
- 仓库根目录新增 `RUNBOOK.md`，统一记录测试、提取、归档、模型资料、预览和正式发布流程。

## 网站与视觉升级

- 全新首页首屏、模型聚光区和合并排行榜。
- 全新模型详情页，更清晰地区分正式模型名与实际 GGUF 文件名。
- 为关键分数和专项成绩加入图形化长条，便于快速比较。
- 加入本地模型 LOGO、BenchLocal-Results 品牌 LOGO 和测试方法插图。
- 增加毛玻璃、柔和发光、立体倾斜和模型 LOGO 椭圆环动效。
- 动效支持悬停缓停，并尊重系统的减少动态效果偏好。
- 统一中英文切换、键盘可访问性、响应式布局和交互反馈。
- 所有运行时字体、图片、样式和脚本均由站点本地提供，不依赖远程静态资源。

## 模型来源与链接

- 模型来源信息按具体模型 slug 维护。
- 能确认具体测试制品时显示“测试模型链接”，同时保留可确认的上游模型链接。
- 仅在发布者为 SC117 或发布者信息确有必要时显示发布者入口。
- 无法确认的制品链接保持为空，不使用宽泛作者主页或猜测链接代替。

## 兼容性

- 旧的思考版详情页地址保留为兼容跳转，不会直接丢失既有链接。
- 历史运行结果继续保存在原始归档中，规范化快照不覆盖原始 summary。
- 站点仍为纯静态 HTML / CSS / JavaScript，可直接通过 GitHub Pages 托管。
- canonical URL、favicon、主题色和外部新窗口安全属性已统一补齐。

## 发布前验证结果

- 23 份配置快照通过数据校验。
- 15 张模型卡片、22 个模型详情或兼容页面、24 个 HTML 文件通过站点校验。
- 4 项单元测试通过。
- 生成数据、首页和详情页与快照保持一致。
- 运行时远程静态资源引用为 0。
- README 展示图已替换为 V1.0 页面截图，旧版截图已移出正式站点目录并保留归档。

## 已知限制

- 结果来自 RTX 5070 Ti 16GB、128GB RAM 和当前 llama.cpp 环境，只适合本项目内部的相对比较。
- 推理速度会受量化、上下文、后端、卸载策略和硬件影响，不应直接作为跨环境性能结论。
- 外部模型来源可能受到 Hugging Face 限流或上游页面变更影响；构建不会因临时限流失败。
- V1.0 使用现有数据完成了迁移和发布门禁验证；使用一轮全新模型结果演练完整采集流程计划在 V1.1 完成。
